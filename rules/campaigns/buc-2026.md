## Cookie: `_join_inputValues`

Used to identify users who arrived from the BUC landing page.

**Format:** Base64-encoded JSON. Set BEFORE visiting `/join`.

```js
// Minimal — enrollment + not_eligible modal only:
const payload = btoa(JSON.stringify({ submission_uid: crypto.randomUUID() }));
document.cookie = `_join_inputValues=${payload}; domain=.seeking.com; path=/`;

// Full — enrollment + join form auto-population (Req.4):
const payload = btoa(JSON.stringify({
  submission_uid: crypto.randomUUID(),
  sex: "3",                        // "3"=Male, "4"=Female (API attribute ID)
  gender_preference: ["248"],      // API preference IDs
  email: "testpilot_<ts>@seeking-test.com",
  dob: 946684800,                  // Unix timestamp (seconds) — NOT ISO string
  account_type: "1"                // optional
}));
document.cookie = `_join_inputValues=${payload}; domain=.seeking.com; path=/`;
```

> ⚠️ `submission_uid: "test-uid-" + Date.now()` is **invalid** — BE validates UUID format. Always use `crypto.randomUUID()`.

**Field names source**: `FE_REPO/resources/react-app/components/auth/utils.tsx` lines 276–417

**Detection logic (`cookieDetector.ts`):**
- Cookie is read and base64-decoded
- Must have a non-empty `submission_uid` string field
- If valid → `detected: true` → dispatches `NOT_ELIGIBLE` modal for existing members

**Cleanup:**
- On `/join` page load: cookie removed by FE after parsing (`Join.tsx` line 219)
- On `not_eligible` modal: cookie removed only on **explicit button click** (Continue or Close) — NOT on auto-dismiss

## Modal Names & Triggers

All modals are Redux-driven via: `state.user.breakup_campaign_2026.campaign_modal`

| Modal Name (code) | Label | Trigger Source | Route Restriction |
|---|---|---|---|
| `not_eligible` | Offer Not Valid | Cookie (`_join_inputValues`) | None |
| `boost_redemption_successful` | Boost Redemption | Backend (`breakup_campaign_2026.campaign_modal`) | None |
| `offer_pending_approval` | Billing Page Modal | Backend | **`/billing/memberships` only** |
| `gold_redemption_successful` | Gold Redemption Success | Backend | None |
| `gold_redemption_extended` | Gold Already Purchased | Backend | None |
| `offer_expired` | Offer Expired | Backend | None |
| `profile_completed` | Profile Completed | Backend | None |

## Modal Selector

```ts
// Redux state path:
state.user.active_campaign          // = 'breakup_campaign_2026'
state.user.breakup_campaign_2026.campaign_modal.name     // modal name
state.user.breakup_campaign_2026.campaign_modal.duration // e.g. 3 (for boosts), 90 (for days)
```

## Close/Action Behavior

- **Cookie-triggered (`not_eligible`):** removes `_join_inputValues` cookie, clears Redux modal
- **Backend-triggered:** calls `setCampaignModalAction(modalName)` API endpoint to mark as seen, then clears Redux modal
